Document Metadata (YAML Header)

The YAML header goes at the very top between --- markers:

---
title: "My Document"
author: "Your Name"
date: "2026-01-21"
format: 
  html:
    toc: true              # Table of contents
    toc-depth: 2           # Heading levels in TOC
    code-fold: true        # Collapsible code
    theme: cosmo           # Bootstrap theme
  pdf:
    documentclass: article
    geometry: margin=1in
bibliography: references.bib
execute:
  echo: true              # Show code by default
  warning: false          # Hide warnings
  message: false          # Hide messages
---

Math Equations

Inline Math

Use single dollar signs: \(E = mc^2\) or \(\alpha + \beta = \gamma\)

Display Math

Use double dollar signs:

\[ f(x) = \int_{-\infty}^{\infty} e^{-x^2} dx \]

\[ \bar{X} = \frac{1}{n}\sum_{i=1}^{n} X_i \]

Common LaTeX Math

Greek letters: $\alpha, \beta, \gamma, \Delta, \Omega$
Subscripts: $x_i, x_{ij}$
Superscripts: $x^2, x^{n+1}$
Fractions: $\frac{a}{b}$
Square root: $\sqrt{x}$
Sum: $\sum_{i=1}^{n} x_i$
Product: $\prod_{i=1}^{n} x_i$
Integral: $\int_a^b f(x)dx$
Limit: $\lim_{x \to \infty} f(x)$
Matrices: $\begin{pmatrix} a & b \\ c & d \end{pmatrix}$

Citations and Bibliography

Citing Sources

Bibliography File

Create references.bib file:

@article{citation2023,
  title = {Article Title},
  author = {Author Name},
  journal = {Journal Name},
  year = {2023},
  volume = {10},
  pages = {1--20}
}

Add to YAML header:

bibliography: references.bib

Keyboard Shortcuts

Common shortcuts in RStudio:

  • Render document: Ctrl + Shift + K
  • Insert chunk: Ctrl + Alt + I
  • Run current chunk: Ctrl + Shift + Enter
  • Run current line: Ctrl + Enter

Best Practices

  1. Use meaningful chunk labels for cross-referencing
  2. Set global options in YAML to avoid repetition
  3. Cache expensive computations with cache: true
  4. Use here package for file paths: here::here("data/file.csv")
  5. Version control your .qmd files (not rendered outputs)
  6. Separate data cleaning from analysis in different chunks
  7. Comment your code for future reference
  8. Test rendering frequently to catch errors early
